📚Instruction Set Reference

Data Movement
  • MOVE

    Move the contents of the source to the destination location. This is a copy operation; the source is not affected.

  • MOVEA

    Move the contents of the source to the destination address register. If a word, it is sign-extended to a longword.

  • MOVEQ

    Move an 8-bit immediate value (sign-extended to 32 bits) into a data register. Faster than MOVE for small constants.

  • MOVEM

    Move multiple registers to or from consecutive memory locations. Used for saving/restoring registers on the stack.

  • MOVEP

    Move data between a data register and alternate bytes in memory. Used for peripheral access on 8-bit buses.

  • LEA

    Load effective address. Computes the effective address of the source operand and loads it into an address register.

  • PEA

    Push effective address. Computes the effective address of the source operand and pushes it onto the stack.

  • CLR

    The destination is cleared and loaded with all zeros. Cannot be used to clear an address register.

  • EXG

    Exchange the contents of two registers. The entire 32-bit contents are exchanged. Supports data, address, and mixed exchanges.

  • SWAP

    Exchange the upper and lower 16-bit words of a data register.

Integer Arithmetic
  • ADD

    Add the source operand to the destination operand and store the result in the destination location.

  • ADDA

    Add the source operand to the destination address register and store the result. The source is sign-extended before adding.

  • ADDI

    Add immediate data to the destination operand. Store the result in the destination operand.

  • ADDQ

    Add a small immediate value (1–8) to the destination operand. Optimized for quick additions.

  • ADDX

    Add the source operand and the extend bit (X) to the destination operand. Used for multi-precision arithmetic.

  • SUB

    Subtract the source operand from the destination operand and store the result in the destination location.

  • SUBA

    Subtract the source operand from the destination address register. Word operations are sign-extended to 32 bits prior to subtraction.

  • SUBI

    Subtract the immediate data from the destination operand. Store the result in the destination operand.

  • SUBQ

    Subtract a small immediate value (1–8) from the destination operand. Optimized for quick subtractions.

  • SUBX

    Subtract the source operand and the extend bit (X) from the destination operand. Used for multi-precision arithmetic.

  • MULS

    Multiply two signed 16-bit values and produce a signed 32-bit result in the destination data register.

  • MULU

    Multiply two unsigned 16-bit values and produce an unsigned 32-bit result in the destination data register.

  • DIVS

    Divide a signed 32-bit dividend by a signed 16-bit divisor. Quotient in lower word, remainder in upper word.

  • DIVU

    Divide an unsigned 32-bit dividend by an unsigned 16-bit divisor. Quotient in lower word, remainder in upper word.

  • NEG

    Subtract the destination operand from 0 and store the result in the destination location. Performs twos complement negation.

  • NEGX

    Subtract the destination operand and the extend bit (X) from 0. Used for multi-precision negation.

  • EXT

    Extend the least-significant byte in a data register to a word, or the least-significant word to a longword.

  • CLR

    The destination is cleared and loaded with all zeros. Cannot be used to clear an address register.

  • CMP

    Subtract the source operand from the destination operand and set the condition codes accordingly. The destination is not modified.

  • CMPA

    Subtract the source operand from the destination address register and set the condition codes. Supports word or longword operations.

  • CMPI

    Subtract the immediate data from the destination operand and set the condition codes accordingly. The destination is not modified.

  • CMPM

    Compare memory to memory using postincrement addressing. Sets condition codes based on the subtraction of source from destination.

  • TST

    Compare the operand with zero. No result is saved, but the CCR bits are set according to the result.

Logical Operations
  • AND

    AND the source operand to the destination operand and store the result in the destination location.

  • ANDI

    AND the immediate data to the destination operand.

  • OR

    OR the source operand to the destination operand, and store the result in the destination location.

  • ORI

    OR the immediate data with the destination operand. Store the result in the destination operand.

  • EOR

    EOR (exclusive or) the source operand with the destination operand and store the result in the destination location.

  • EORI

    EOR the immediate data with the contents of the destination operand. Store the result in the destination operand.

  • NOT

    Calculate the logical complement of the destination and store the result. NOT performs bit-by-bit logical complementation.

Shift & Rotate Operations
  • ASL

    Arithmetically shift the bits left. The shift count may be specified as a literal, register contents, or the value 1.

  • ASR

    Arithmetically shift the bits right. The shift count may be specified as a literal, register contents, or the value 1.

  • LSL

    Logically shift the bits left. A zero is shifted into the input position. The shifted-out bit is copied into the C and X bits of the CCR.

  • LSR

    Logically shift the bits right. A zero is shifted into the input position. The shifted-out bit is copied into the C and X bits of the CCR.

  • ROL

    Rotate the bits left. The extend bit X is not included in the operation. Bits shifted out at one end are shifted into the other end.

  • ROR

    Rotate the bits right. The extend bit X is not included in the operation. Bits shifted out at one end are shifted into the other end.

  • ROXL

    Rotate left through the extend bit (X). The X bit is included in the rotation as an additional bit position.

  • ROXR

    Rotate right through the extend bit (X). The X bit is included in the rotation as an additional bit position.

Bit Manipulation
  • BTST

    Test a bit in the destination operand. The Z flag is set if the tested bit is zero, cleared otherwise.

  • BSET

    Test a bit in the destination operand and then set it. The Z flag reflects the original value of the bit before setting.

  • BCLR

    Test a bit in the destination operand and then clear it. The Z flag reflects the original value of the bit before clearing.

  • BCHG

    Test a bit in the destination operand and then toggle (complement) it. The Z flag reflects the original value of the bit.

Condition Code Register Operations
  • ANDI to CCR

    AND immediate data with the condition code register (CCR). Used to selectively clear CCR bits.

  • ORI to CCR

    OR immediate data with the condition code register (CCR). Used to selectively set CCR bits.

  • EORI to CCR

    Exclusive-OR immediate data with the condition code register (CCR). Used to selectively toggle CCR bits.

  • MOVE to CCR

    Move the source operand to the condition code register. The upper byte of the SR is not affected.

  • MOVE from CCR

    Move the contents of the condition code register to the destination operand.

Program Control & Branching
  • BRA

    Branch always. Program execution continues at [PC] + displacement. Displacement is a twos complement value (8-bit or 16-bit).

  • BSR

    Branch to subroutine. Pushes the next instruction address onto the stack (A7). Execution continues at [PC] + displacement.

  • Bcc

    Branch conditionally. Includes BHI, BLS, BCC, BCS, BNE, BEQ, BVC, BVS, BPL, BMI, BGE, BLT, BGT, BLE. Branch is taken if the specified condition is true.

  • DBcc

    Test condition, decrement, and branch. Includes DBHI, DBLS, DBCC, DBCS, DBNE, DBEQ, DBVC, DBVS, DBPL, DBMI, DBGE, DBLT, DBGT, DBLE, DBF, DBT. If condition is false, decrement the data register and branch if result is not −1.

  • Scc

    Set according to condition. Includes SHI, SLS, SCC, SCS, SNE, SEQ, SVC, SVS, SPL, SMI, SGE, SLT, SGT, SLE, SF, ST. Sets the destination byte to all ones if condition is true, all zeros otherwise.

  • JMP

    Program execution continues at the effective address specified by the instruction.

  • JSR

    Jump to subroutine. Pushes the longword address of the next instruction onto the system stack (A7). Execution continues at the specified address.

  • RTS

    Return from subroutine. The program counter is pulled from the stack. Used to terminate a subroutine and return to the calling code.

  • RTR

    Return and restore condition codes. Pulls the CCR and then the PC from the stack.

  • RTD

    Return and deallocate. Pulls the PC from the stack and then adds a displacement to the stack pointer to deallocate stack frame parameters.

System Control & Exception Handling
  • RESET

    Assert the RESET line. In supervisor mode, resets external devices without affecting internal processor state.

  • NOP

    No operation. Performs no action other than advancing the program counter. Used for timing or alignment.

  • STOP

    Load immediate data into the status register and stop execution. The processor waits for an interrupt or reset.

  • RTE

    Return from exception. Restores the status register and program counter from the supervisor stack.

  • TRAP

    Initiate a software trap exception. The trap number (0–15) selects the vector for the exception handler.

  • TRAPV

    Trap on overflow. If the overflow flag (V) is set, a TRAPV exception is generated.

  • CHK

    Check register against bounds. If the data register value is negative or greater than the upper bound, a CHK exception is generated.

  • LINK

    Link and allocate. Pushes the specified address register onto the stack, copies SP to the address register, then adds a displacement to SP to allocate a stack frame.

  • UNLK

    Unlink. Restores SP from the address register and then pops the address register from the stack. Reverses the LINK operation.

  • MOVE to SR

    Move the source operand to the status register. Privileged instruction; affects both the CCR and supervisor byte.

  • MOVE from SR

    Move the contents of the status register to the destination operand.

  • ORI to SR

    OR immediate data with the status register. Privileged instruction used to set SR bits.

  • ANDI to SR

    AND immediate data with the status register. Privileged instruction used to clear SR bits.

  • EORI to SR

    Exclusive-OR immediate data with the status register. Privileged instruction used to toggle SR bits.

  • TAS

    Test and set. Tests the destination byte (sets N and Z flags), then sets the high-order bit (bit 7) of the operand. Uses an indivisible read-modify-write bus cycle.

💡Tips & Tricks

⏱ Time Delay

You can add delays in your M68K assembly code using simple loops or dedicated delay routines. This is useful for controlling execution timing and creating pauses between operations. Perfect for simulating real-world timing behavior in your emulator testing.

🔧 Toggle Advanced Registers

Enable the advanced register display to view detailed register states beyond standard data and address registers. This includes condition codes, status register flags, and internal emulator state. Essential for debugging complex programs that depend on specific flag conditions.
